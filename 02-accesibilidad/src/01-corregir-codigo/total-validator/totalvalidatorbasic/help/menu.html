File Menu

With the Basic tool there is a 'Save' option to save your current workspace. But with the Pro tool there are all the usual options for creating a new workspace (New), opening an existing workspace (Open), saving the current workspace (Save and Save As), and deleting the current workspace (Delete). There is also a list of the last 10 workspaces used to select from.

You can also create your own default settings that are used whenever you create a new workspace, or use the Reset buttons. Simply save your workspace with the name of 'defaults' and any settings this contains will then become the new defaults.

The File menu also contains the following options: 'Validate' starts a validation. 'Reset List' clears the Starting web page dropdown on the Main Tab, 'Reset All' resets most options to their defaults, and 'Exit' closes the Tool.

Options - Edit meta names (Pro tool only)

For HTML5 documents, the 'name' attribute in the <meta> element is restricted to values documented in the HTML5 specification with extensions in the WHATWG Wiki. Each Total Validator release contains the latest set of valid values at that time. So Total Validator will not recognise any changes to the wiki after the release date, and may mark some 'name' values as incorrect.

You could wait until the next release of Total Validator to automatically pick up any new names that you use, or you can use the 'Edit meta names' option to add them yourself now. But note that any changes you make will be automatically overwritten when you run a later release.

Options - Check for Updates

When you use this option the tool will try to contact the Total Validator site to see if there is a later version of the tool available.

If there is a more up to date version you will be prompted with a link that takes you to the appropriate update page.

top

Options - Auto-update Check

If you select this option the tool will check for updates automatically each time you start the tool.

Note that this is a completely silent check, so if the tool cannot contact the update site for any reason (such as a blocking proxy server) then no message will be displayed and the tool will start in the normal way. So it's a good idea to perform a manual test using the previous option first.

Options - Use System Proxy

By default this option is selected and will attempt to use any operating system proxy server settings: For example, the Windows 'Internet Options' control panel applet settings.

At this time auto-proxy configuration scripts are ignored, but with the Pro tool you can supply the proxy server details on the Network tab instead.

Note that when you change this option you must restart the tool for it to have any effect.

Options - Splashscreen

This option determines whether a splash screen displaying details of the current version is displayed at startup.

Options - SSL Bug Fix

If you are running Total Validator with the latest version of Java and have SSL errors when validating pages or checking links, the issue may be due to a known bug in Java. This problem is known to affect sites hosted on AWS but may affect other sites as well.

Selecting this option may resolve the problem. But a side-affect is that sites using self-certified certificates may have SSL errors when validating them. So only use this option when you have a problem with a specific site, and leave it turned off otherwise.

Note that when you change this option you must restart the tool for it to have any effect.

Help Menu

There are four options on this menu:

  • The 'Help' option displays this help
  • The 'Updates' option (Pro tool) displays information about updates and subscriptions to the Pro tool
  • The 'Upgrade' option (Basic tool) displays information about purchasing the Pro tool
  • The 'Change log' option displays the online changes log
  • The 'About' option displays the version and build details of the tool, various paths used by the tool, and allows you to turn on debugging when requested by Support.
